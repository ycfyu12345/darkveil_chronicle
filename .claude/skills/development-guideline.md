# Darkveil Chronicle - 开发准则

## 核心原则

**每完成一个任务并通过验证后，必须立即提交 git，不得合并到后续任务中。**

---

## 任务流程

### 1. 任务开始
- 从 `task_breakdown.md` 中选取一个待完成的任务
- 确认该任务的依赖已全部完成

### 2. 任务执行
- 按照任务分解中的"操作"列实现功能
- 不要实现任务范围以外的功能

### 3. 任务验证
- 按照任务分解中的"验证"列进行验证
- 必须在 Godot 编辑器中运行测试
- 验证通过后才能进入下一步

### 4. Git 提交（强制）
**任务验证通过后，必须立即提交：**

```bash
git add <相关文件>
git commit -m "$(cat <<'EOF'
<任务ID>: <简短描述>

<任务ID>: <简短描述>
- 完成内容1
- 完成内容2

验证方式: <如何验证此任务>
EOF
)"
```

### 5. 提交示例

假设完成 T1.1 后：

```bash
git add autoload/game_manager.gd
git commit -m "$(cat <<'EOF'
T1.1: 实现 GameManager autoload

- 添加 game_state 状态管理
- 添加 gold 金币系统
- 添加 party 队伍管理
- 添加 current_battle_setup 战斗上下文
- 实现 change_scene() 场景切换

验证方式: Engine.get_singleton("GameManager") 可获取实例
EOF
)"
```

---

## 提交信息规范

### 格式
```
<任务ID>: <简短描述>

<详细说明（可选）>

验证方式: <如何验证此任务>
```

### 规则
- 标题不超过 72 字符
- 使用中文撰写提交信息
- 任务 ID 使用 `T{x.y}` 格式
- 每个提交只包含一个任务

### 示例
```
T1.1: 实现 GameManager autoload

T1.2: 实现 SignalManager 全局信号总线

T2.3: 实现 BattleLogic 伤害计算
```

---

## 禁止事项

- **禁止**将多个任务合并到一次提交
- **禁止**完成任务后不提交就去实现下一个任务
- **禁止**提交包含未完成功能的代码
- **禁止**提交无法通过验证的代码

---

## 任务状态追踪（强制）

每完成一个任务并验证通过后，**必须同时完成两件事**：

### 1. 在 task_breakdown.md 中标记任务完成

找到对应的任务行，将验证列更新为：

```
- [x] <验证方式和结果>
```

示例：
```markdown
| T1.1 | `autoload/game_manager.gd` | ... | - [x] Engine.get_singleton("GameManager") 可获取实例 |
```

### 2. 提交 git

按照上面的规范提交代码。

### 标记时机

- **必须**在 git commit 之前完成标记
- **必须**在验证通过后立即标记
- 标记和提交是同一个动作的两个步骤

---

## 问题记录（强制）

开发和测试过程中遇到问题时，**必须立即记录**，不得只停留在脑中或口头。

### 记录位置

在 `task_breakdown.md` 底部创建 `## 问题记录` 章节：

```markdown
## 问题记录

### 待解决
| 日期 | 任务 | 问题描述 | 影响 |
|------|------|----------|------|
| 2026-04-09 | T2.3 | calculate_damage 速度平局时 randi() 结果不稳定 | 可能导致单元测试偶尔失败 |

### 已解决
| 日期 | 任务 | 问题描述 | 解决方案 |
|------|------|----------|----------|
| 2026-04-09 | T1.5 | AudioManager 空实现导致报错 | 添加 if sfx_name in _cached_sfx: return |
```

### 记录时机

遇到以下情况时，必须立即记录：
- 遇到无法立即解决的 bug
- 发现设计不合理需要后续重构
- 遇到技术难点需要进一步研究
- 依赖外部条件（设计稿、素材等）
- 某个问题花费超过 30 分钟仍未解决

### 处理原则

1. **立即记录**：不要等到问题解决才记录，先记下来再尝试解决
2. **定期回顾**：每次开发开始前，查看问题记录
3. **解决闭环**：问题解决后，移到"已解决"区并记录解决方案
4. **不要堆积**：如果一个问题超过 2 天仍未解决，必须升级反馈

```bash
git log --oneline
git log --oneline -10  # 最近10条
```
